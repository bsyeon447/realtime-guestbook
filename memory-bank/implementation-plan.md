# 구현 계획

## 완료된 구현 범위

1.  **프로젝트 환경 설정**
    *   `server`와 `client` 디렉토리를 분리했습니다.
    *   백엔드는 Express, Socket.io, multer 기반으로 구성했습니다.
    *   프론트엔드는 React와 Vite, socket.io-client 기반으로 구성했습니다.

2.  **핵심 백엔드 기능 구현**
    *   `server/data.json` 파일을 사용하여 `{ posts: [] }` 형태로 방명록과 댓글을 보존합니다.
    *   `server/uploads` 폴더에 이미지 업로드 파일을 저장합니다.
    *   `GET /api/posts`, `POST /api/posts`, `GET /api/health` 엔드포인트를 제공합니다.
    *   Socket.io로 초기 목록, 새 방명록, 상세 댓글 룸 입장/퇴장, 댓글 추가, 포스트 갱신 이벤트를 처리합니다.

3.  **핵심 프론트엔드 기능 구현**
    *   작성 폼, HTML5 Canvas 그림판, 포스트잇 그리드, 상세 모달, 댓글 UI를 구현했습니다.
    *   사진 업로드와 직접 그린 그림 중 하나를 선택하여 등록할 수 있습니다.
    *   Socket.io 이벤트를 수신하여 새 방명록과 댓글을 새로고침 없이 반영합니다.

4.  **문서화 및 검증**
    *   README에 서버/클라이언트 실행 방법과 주요 기능을 문서화했습니다.
    *   서버 문법과 JSON 설정을 검증했습니다. npm 레지스트리 접근은 403 정책 제한으로 설치/빌드 검증을 완료하지 못했습니다.

## 향후 확장 제안

*   사용자 이름, 비밀번호 또는 관리자 삭제 기능 추가
*   MongoDB/SQLite 기반 영구 저장소 전환
*   이미지 리사이징 및 MIME/확장자 검증 강화
*   E2E 테스트와 API 테스트 자동화
