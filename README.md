# 실시간 방명록 (Realtime Guestbook)

Supabase와 React를 사용하여 제작된 실시간 방명록 애플리케이션입니다. 사용자는 텍스트와 함께 직접 그린 그림이나 사진을 방명록에 남길 수 있으며, 모든 변경 사항은 페이지 새로고침 없이 실시간으로 반영됩니다.

## 주요 기능

- **실시간 방명록**: 새로운 방명록과 댓글이 실시간으로 모든 사용자에게 표시됩니다.
- **이미지 업로드**: 원하는 사진을 방명록에 첨부할 수 있습니다.
- **그림판**: 캔버스에 직접 그림을 그려 등록할 수 있습니다.
- **상세 보기**: 각 방명록을 클릭하여 상세 내용과 댓글을 확인할 수 있습니다.

## 기술 스택

- **프론트엔드**: React, 
eact-konva
- **백엔드 (BaaS)**: Supabase
  - **Database**: PostgreSQL
  - **Storage**: 이미지 파일 저장
  - **Realtime**: 데이터 변경 실시간 감지 및 브로드캐스트
- **스타일링**: CSS (또는 추후 적용될 styled-components)

## 시작하기

### 1. Supabase 프로젝트 설정

1.  Supabase에 가입하고 새 프로젝트를 생성합니다.
2.  **SQL Editor**에서 아래의 쿼리를 실행하여 테이블과 스토리지 버킷을 생성합니다.

    `sql
    -- posts 테이블 생성
    CREATE TABLE posts (
      id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL,
      message text,
      image_url text
    );

    -- comments 테이블 생성
    CREATE TABLE comments (
      id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL,
      content text,
      post_id bigint REFERENCES posts(id) ON DELETE CASCADE
    );

    -- 스토리지 버킷 생성
    INSERT INTO storage.buckets (id, name, public) VALUES ('guestbook-images', 'guestbook-images', true);

    -- 테이블에 대한 Realtime 활성화
    -- (Supabase 대시보드의 Database -> Replication 에서 설정할 수도 있습니다.)
    ALTER PUBLICATION supabase_realtime ADD TABLE posts, comments;
    `

3.  **Settings -> API** 에서 프로젝트 URL과 non 키를 확인합니다.

### 2. 로컬 환경 설정

1.  이 저장소를 클론합니다.
    `bash
    git clone https://github.com/your-username/realtime-guestbook.git
    cd realtime-guestbook/client
    `

2.  client 디렉토리에 .env.local 파일을 생성하고 Supabase에서 확인한 키를 입력합니다.
    `
    REACT_APP_SUPABASE_URL="YOUR_SUPABASE_URL"
    REACT_APP_SUPABASE_ANON_KEY="YOUR_SUPABASE_ANON_KEY"
    `

3.  의존성을 설치하고 개발 서버를 시작합니다.
    `bash
    npm install
    npm start
    `

4.  브라우저에서 http://localhost:3000으로 접속합니다.
